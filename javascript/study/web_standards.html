웹 표준을 위한 가이드라인

W3C 웹표준

W3C의 설립 취지는 웹의 지속적인 성장을 도모하는 프로토콜과 가이드라인을 개발하여 월드 와이드 웹의 모든 잠재력을 이끌어 내는 것이며, 설립 목적인 웹 표준과 가이드라인 개발을 수행하고 지금까지의 결과로 지난 10년간 80여개의 W3C 권고안을 발표하였습니다.
그 덕에 많은 브라우저들이 탄생했음에도 W3C의 활동으로 인해 규칙이 정해져 통일된 웹 브라우저 환경이 조성되었습니다.

웹 표준

웹 표준이란?

1. 웹에서 표준적으로 사용되는 기술 또는 규칙
2. 표준화 단체인 w3c가 권고한 표준안에 따라 웹사이트를 작성해야한다.
3. 표준 스펙을 잘 지키는 것 뿐만 아니라 구조적 마크업과 표현 레이아웃 및 사용자 행쥐 제어를 잘 분리하는 구축 방식이다.

중요한 이유?

1. 지금같이 표준이 있었더라면 한번만 작업해도 되고 많은 노동력과 시간을 아낄 수 있으며, 자연스레 정보 제공의 질도 올라갈 것

웹표준의 장점?

1. 개발&운영 효율성이 좋아지며 관리가 용이하다
2. 다양한 프로그램으로의 접근성이 향상, 모든 연령을 커버 가능케 한다.
3. 보조 기기를 사용하는 사용자들이 정보 습득에 있어서 정확성을 향상 시킬 수 있다.
4. CSS와 HTML이 분리되어서 유비보수 및 불필요한 요소가 줄어들어 속도향상에 도움이 된다.
5. 검색엔진 최적화가 가능하다

웹 표준을 준수하고 있는지 확인해보기

하단의 사이트를 통해 자신이 알아보고자 하는 웹 페이지가 웹표준을 준수하고 있는지 확인해볼 수 있으며, 웹사이트 이름에 마크업이라는 단어가 나왔는데 w3c의 웹 표준대로 html 코딩이 되었는가를 알려주는 사이트입니다.

W3C Markup 검증 서비스 : http://validator.kldp.org

웹 접근성

웹 접근성이란?

웹 접근성이란 「지능정보화기본법」에 따라 장애인이나 고령자분들이 웹 사이트에서 제공하는 정보를 장애인과 고령자 등등 동등하게 접근하고 이용 할 수 있도록 보장하는 것으로 웹 접근성 준수는 법적의무사항 이며 목적,인적,환경적 요인에 제약 없이 모든 웹 정보에 접근이 가능해야 합니다.

웹 접근성 지침 2.1

한국형 웹 콘텐츠 접근성 지침 2.1은 4가지 원칙과 각 원칙을 준수하기 위한 13개 지침 및 해당 지침의 준수여부를 확인하기 위해 24개의 검사항목으로 구성되어 있습니다.

원칙 1. 인식의 용이성 (Perceivable) : 모든 콘텐츠는 사용자가 인식할 수 있어야 한다.
원칙 2. 운용의 용이성(Operable) : 사용자 인터페이스 구성요소는 조작 가능하고 내비게이션 할 수 있어야 한다.
원칙 3. 이해의 용이성(Understandable) : 콘텐츠는 이해할 수 있어야 한다.
원칙 4. 견고성(Robust) : 웹 콘텐츠는 미래의 기술로도 접근할 수 있도록 견고하게 만들어야 한다.

웹 호환성

웹 호환성이란?

1. 웹 표준을 준수해 브라우저간의 호환성을 유지시킵니다.
2. 어느 웹 브라우저를 사용하던 브라우저간에 출력되는 결과가 동일하게 나오게끔 합니다.
3. 표준 웹 기술을 활용해 최대한 공통된 요소를 사용해서 웹페이지를 제작하고 어느 한쪽만의 요소를 사용하지 않아야 합니다.
4. 최종 목적은 웹 브라우저의 종류와 버전에 상관없이 웹사이트 접근이 유효하도록 하는것입니다.

크로스 브라우징

크로스 브라우징이란 "각 플랫폼에 맞춰 실체화 되는 기술을 비슷하게 만들어 모든 브라우저가 최대한 동등하게 최적화 되어 공통된 요소를 사용하여 웹페이지를 만드는 기법입니다."
즉 어떤 브라우저를 사용하더라도 문제가 발생하지않고 동일한 화면을 보여주는것이 크로스 브라우징이 설명하는 핵심이라고 볼 수 있습니다.
